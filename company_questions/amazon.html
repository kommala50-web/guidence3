Round A — Online Assessment (OA)
Typically 2–3 timed coding tasks (easy → medium)
~60–120 minutes (total)
1) Two Sum (return indices)
Hashmap — O(n)
Solution: Use a map to store seen numbers and their indices. For each number, check complement (target - num) in map.
# Python (clean)
def two_sum(nums, target):
    seen = {}
    for i, n in enumerate(nums):
        comp = target - n
        if comp in seen:
            return [seen[comp], i]
        seen[n] = i
    return []
This is O(n) time and O(n) extra space. Mention edgecases: duplicates and negative numbers.
Tip: Talk through example and complexity before coding; state that you're assuming one valid answer exists (or discuss alternatives).
2) Check if a number is palindrome (without converting to string)
Math / Two-pointer
Solution: Reverse half of the number and compare to remaining half to avoid overflow and extra space.
# Python (reverse half approach)
def is_palindrome(x):
    if x < 0 or (x % 10 == 0 and x != 0): return False
    rev = 0
    while x > rev:
        rev = rev*10 + x % 10
        x //= 10
    return x == rev or x == rev // 10
Avoids converting to string. Works in O(log10(n)) time where number of digits is log10(n).
Tip: Mention constraints (negative numbers not palindrome) and explain why reversing full number may overflow in some languages — so reverse-half is safer.
3) Valid Parentheses
Stack
Solution: Use a stack; on opening push, on closing check top matches.
# Python
def is_valid(s):
    pairs = {')':'(', ']':'[', '}':'{'}
    stack = []
    for ch in s:
        if ch in pairs.values():
            stack.append(ch)
        elif ch in pairs:
            if not stack or stack.pop() != pairs[ch]:
                return False
    return not stack
Handles mixed/nested parentheses. Complexity O(n) time and O(n) space.
Tip: Ask about allowed characters; include tests for edge cases (empty string, odd length, early closing).
4) Remove duplicates from sorted array (in-place)
Two-pointer
Solution: Maintain slow and fast pointers; write unique elements in place.
# Python
def remove_duplicates(nums):
    if not nums: return 0
    j = 0
    for i in range(1, len(nums)):
        if nums[i] != nums[j]:
            j += 1
            nums[j] = nums[i]
    return j + 1  # new length
O(n) time, O(1) extra space. Works because array is already sorted.
Tip: Explain why in-place is required and show example transforms step-by-step.
5) Merge Intervals (given list of intervals)
Sorting + Merge
Solution: Sort by start, then iterate merging overlapping intervals.
# Python
def merge(intervals):
    if not intervals: return []
    intervals.sort(key=lambda x: x[0])
    res = [intervals[0]]
    for s,e in intervals[1:]:
        last_s, last_e = res[-1]
        if s <= last_e:
            res[-1] = (last_s, max(last_e, e))
        else:
            res.append((s,e))
    return res
Complexity O(n log n) due to sort. Mention inclusive/exclusive endpoints if asked.
Tip: Talk about edgecases (single interval, fully nested intervals) and memory vs in-place outputs.
Round B — Technical Round 1 (Coding / DSA)
Deeper algorithmic problems; explain approach & optimize
~45–60 minutes
1) Reverse a singly linked list (iterative & recursive)
Pointers
Iterative Solution: Use prev, curr pointers to reverse links in-place.
# Python (iterative)
class Node:
    def __init__(self,val,next=None):
        self.val = val; self.next = next

def reverse(head):
    prev = None
    curr = head
    while curr:
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt
    return prev  # new head
Iterative is O(n) time, O(1) space. Recursive solution reverses rest of list and attaches head at end — but uses O(n) stack.
Tip: Draw nodes and narrate pointer updates. Mention stack limits for recursion in production environments.
2) Merge K Sorted Lists
Heap / Divide & Conquer
Heap approach: Use a min-heap seeded with each list's head; pop smallest, append to result, push next from that list.
# Python (heapq)
import heapq

def merge_k_lists(lists):
    heap = []
    for i, node in enumerate(lists):
        if node:
            heapq.heappush(heap, (node.val, i, node))
    dummy = Node(0)
    tail = dummy
    while heap:
        val, i, node = heapq.heappop(heap)
        tail.next = node
        tail = tail.next
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))
    return dummy.next
Time O(N log k) where N total nodes, k lists. Note the tuple with index to avoid comparison issues between nodes in heap.
Tip: Mention divide-and-conquer merge (pairwise) as alternative with similar complexity.
3) Longest Increasing Subsequence (O(n log n) approach)
DP + Binary Search
Patience sorting / tails array: Maintain tails array where tails[len] is smallest tail for increasing subsequence of length len+1. Binary search to place each element.
# Python
import bisect

def length_of_lis(nums):
    tails = []
    for x in nums:
        i = bisect.bisect_left(tails, x)
        if i == len(tails):
            tails.append(x)
        else:
            tails[i] = x
    return len(tails)
O(n log n) time. Good to first explain O(n^2) DP for clarity, then optimize to tails approach.
Tip: Explain why tails preserves smallest possible tails for subsequences of each length (invariants).
4) Longest substring without repeating characters
Sliding window
Solution: Maintain last-seen index map and slide window with two pointers.
# Python
def length_of_longest_substring(s):
    last = {}
    left = 0
    best = 0
    for right, ch in enumerate(s):
        if ch in last and last[ch] >= left:
            left = last[ch] + 1
        last[ch] = right
        best = max(best, right - left + 1)
    return best
O(n) time. Clarify character set assumptions (ASCII vs Unicode) and handling multi-byte if necessary.
Tip: Walk through example 'abba' to show left movement nuance.
5) Find median from a data stream
Two Heaps
Solution: Maintain max-heap for lower half and min-heap for upper half; balance sizes.
# Python
import heapq

class MedianFinder:
    def __init__(self):
        self.lo = []  # max-heap (store negatives)
        self.hi = []  # min-heap

    def addNum(self, num):
        heapq.heappush(self.lo, -num)
        if self.lo and self.hi and (-self.lo[0] > self.hi[0]):
            val = -heapq.heappop(self.lo)
            heapq.heappush(self.hi, val)
        if len(self.lo) > len(self.hi) + 1:
            heapq.heappush(self.hi, -heapq.heappop(self.lo))
        if len(self.hi) > len(self.lo):
            heapq.heappush(self.lo, -heapq.heappop(self.hi))

    def findMedian(self):
        if len(self.lo) > len(self.hi):
            return -self.lo[0]
        return (-self.lo[0] + self.hi[0]) / 2
Insert O(log n), findMedian O(1). For huge streams, consider approximate algorithms (t-digest, quantile sketches).
Tip: Demonstrate balancing steps with a small sequence to show invariants.
Round C — Technical Round 2 (System Design / Deep Dive)
High-level design, scaling, trade-offs
~45–75 minutes
1) Design a URL shortening service (scale to millions)
System Design
High-level components: API servers, load balancer, user & url metadata DB (NoSQL), cache (Redis), unique ID generator (base62), redirect service, analytics pipeline.
Clarify requirements (QPS, write-heavy vs read-heavy, TTL, custom aliases). Use consistent hashing or sharding for DB; use replication for HA; use CDNs for global redirect latency; handle collisions with retry; analytics via async pipeline (Kafka → analytics DB).
Tip: Start with simple functional flow, then address bottlenecks, availability, and data retention. Discuss trade-offs: collision-free IDs vs short length; storage vs fast reads.
2) Design an order processing system (inventory & concurrency)
Distributed systems
Approach: API gateway → Order service → Inventory service → Payment service. Use message queue (SQS/Kafka) for asynchronous work and Saga pattern / compensating transactions for distributed consistency.
Key points: optimistic locking/versioning for inventory, reservation at checkout (soft hold), idempotency for retries, handling partial failures (payment success but downstream fail), and monitoring. Use CQRS for read-heavy product catalogs.
Tip: Draw sequence diagram, mention metrics (latency, failure rates), and propose rollback/compensation strategies for failed flows.
3) Design a recommendation system
ML / Architecture
Options: Collaborative filtering, content-based, hybrid. Batch compute with Spark for large-scale model training; online ranking for personalization; caching top-N for low-latency responses.
Discuss features, offline training, online serving, A/B testing, cold-start problem solutions (popularity or content features), latency trade-offs, and personalization ethics (diversity, fairness).
Tip: Propose evaluation metrics (precision@k, recall, CTR) and monitoring for model drift.
4) When to use eventual vs strong consistency?
Design Decision
Guideline: Use strong consistency when correctness matters per operation (payments, inventory) and eventual when availability and latency are prioritized (analytics, recommendations).
Discuss CAP theorem, hybrid approaches (leader for writes, read replicas with read-your-writes), and how to mitigate user-visible inconsistencies (soft locks, versioning).
Tip: Give concrete examples comparing the two and propose mitigation patterns for eventual consistency.
5) Optimize a slow SQL query — example & index choice
DB optimization
Example: Query filtering by user_id and created_at. Use composite index (user_id, created_at) when filtering and ordering by those columns. Use EXPLAIN to inspect plan.
-- Example: create composite index
CREATE INDEX idx_user_created ON orders (user_id, created_at DESC);

-- Use explain to check if index is used:
EXPLAIN SELECT id, total FROM orders WHERE user_id = 123 ORDER BY created_at DESC LIMIT 20;
Consider write cost of indexes and whether partitioning (by date) helps for very large tables. Also consider covering indexes to avoid lookups.
Tip: Always show how your index choice maps to WHERE / ORDER BY clauses and measure with EXPLAIN ANALYZE.
Round D — Managerial / Leadership Principles
Behavioral deep-dive; STAR stories mapped to LPs
~45 minutes
1) Tell me about a time you demonstrated Customer Obsession
Leadership
Structure: Use STAR. Situation → Task → Action → Result, quantify impact.
Example: You discovered a recurring customer issue via support tickets, initiated telemetry, and shipped a fix improving NPS by X points. Show metrics and follow-up actions taken to prevent recurrence.
Tip: Prepare multiple STAR stories and map each to different leadership principles beforehand.
2) Describe a time you had to make a trade-off with limited resources
Decision-making
Key points: Show how you evaluated trade-offs, prioritized customer/business impact, and measured outcomes.
Be explicit about alternatives considered, reasons for the final decision, and how you communicated and monitored results.
Tip: Use quantifiable metrics (throughput, cost saved, delivery time) to strengthen your example.
3) Give an example of 'Dive Deep' — how you investigated a production issue
Problem solving
Suggested structure: Describe telemetry used, hypothesis testing, root-cause, fix, and postmortem actions.
Talk about logs, metrics, and experiments you ran. Show you closed the loop with permanent fixes and monitoring to detect recurrence.
Tip: Emphasize reproducibility (how you ensured the bug wouldn't return) and communication with stakeholders.
4) Tell me about a time you influenced without authority
Leadership / Collaboration
Answer guidance: Show persuasion via data, empathy, and building consensus across teams; include outcome metrics.
Highlight stakeholders, constraints, and how you managed resistance. Show follow-through after agreement.
Tip: Mention one concrete deliverable that changed due to your influence and measurable result.
5) How do you prioritize hiring, mentoring and shipping features?
People & Delivery
Balanced approach: Prioritize based on business impact, team capacity, and long-term health (ownership, mentoring). Use clear metrics for success.
Explain trade-offs: hiring slows immediate feature velocity but increases long-term capacity. Show how you balance sprint plans, mentorship time, and technical debt reduction.
Tip: Provide an example timeline where you balanced these components and the outcome.
Round E — HR / Offer & Logistics
Salary band, relocation, start date, and final checks
~15–30 minutes
1) What are your salary expectations?
Logistics
Be honest and give a researched range. You can say you are open to discussion depending on the full compensation package and role level.
Recruiters often have bands — asking them for the band can help align expectations. Include bonuses, equity, and location cost-of-living when thinking of total comp.
Tip: Provide a range rather than a single number, and mention flexibility for the right opportunity.
2) When can you join / notice period?
Logistics
State your current notice period and any constraints clearly. If negotiating start date, propose possible windows you can join in.
Recruiters need to plan hiring timelines. Be transparent about any obligations or dependent offers.
Tip: If you need to negotiate, propose a mutually agreeable timeline and explain reasons briefly.
3) Any visa or relocation constraints?
Logistics
Be upfront about visa sponsorship needs or relocation constraints to avoid surprises later.
Some roles have restrictions; early transparency helps the recruiter guide options (local role, remote, or relocation support).
Tip: Provide clear timelines (e.g., "I require H-1 transfer, currently on X visa") if relevant.
4) Are there any conflicts of interest or NDAs?
Legal / Compliance
Answer honestly. If you have NDAs, explain constraints at a high level without breaking confidentiality.
The company may need to route legal checks or discuss public work restrictions; be open.
Tip: If unsure, say you can share details with recruiter or legal team and provide contact for your current employer if necessary.
5) Do you have any questions for us?
Closing
Good questions: team structure, success metrics for the role, ramp plan, interview feedback timeline, and next steps.
Shows interest and helps you evaluate fit. Ask about team culture and immediate priorities for the role.
Tip: Prepare 4–6 questions and prioritize them based on what's most important to you.
