⚡ JavaScript Full Roadmap

Master JavaScript from basics to advanced — clear explanations + runnable examples.

← Back to Skills Download PDF Video Resource

1. Introduction to JavaScript

Description: JavaScript is the language of the web — it enables dynamic behavior, DOM manipulation, and interaction with web APIs. It runs in the browser and on servers via Node.js.

<script>
console.log("Hello JavaScript — welcome!");
</script>

2. Syntax & Variables

Description: JavaScript statements end with semicolons (optional). Declare variables with let, const, or legacy var. Use const for values that shouldn't change.

<script>
let name = "Ramesh";
const PI = 3.14159;
var legacy = "old";
console.log(name, PI, legacy);
</script>

3. Data Types

Description: Primitive types: string, number, boolean, null, undefined, symbol, bigint. Complex: object (including arrays & functions).

<script>
let s = "text"; // string
let n = 100;    // number
let b = true;   // boolean
let o = {x:1};  // object
let a = [1,2];  // array
</script>

4. Operators

Description: Arithmetic (+ - * / %), comparison (==, ===, !=, !==, >, <=), logical (&&, ||, !), ternary (cond ? a : b), and assignment operators exist.

<script>
let x = 5 + 3;
let same = (5 == '5');     // true (loose)
let strict = (5 === '5');  // false (strict)
let result = x > 7 ? 'big' : 'small';
</script>

5. Conditional Statements

Description: Control logic with if/else and switch. Use strict equality (===) to avoid type coercion surprises.

<script>
let score = 78;
if(score >= 90) { console.log("A"); }
else if(score >= 75) { console.log("B"); }
else { console.log("C"); }

let day = 2;
switch(day){
  case 1: console.log("Mon"); break;
  case 2: console.log("Tue"); break;
}
</script>

6. Loops

Description: Repeat blocks with for, while, do...while. Use for...of to iterate arrays and for...in for object keys (careful: inherited keys).

<script>
for(let i=0;i<3;i++){ console.log(i); }
let arr=[10,20];
for(let v of arr){ console.log(v); }
</script>

7. Functions

Description: Functions package reusable logic. They can be declared (hoisted) or expressed (not hoisted the same way). Use default parameters when helpful.

<script>
function add(a=0,b=0){
  return a + b;
}
const sub = function(a,b){ return a-b; };
console.log(add(2,3), sub(5,2));
</script>

8. Arrow Functions

Description: Compact function syntax from ES6. Arrow functions do not have their own this, arguments, or new.target — useful for short callbacks.

<script>
const nums = [1,2,3];
const doubled = nums.map(n => n * 2);
console.log(doubled);
</script>

9. Scope & Hoisting

Description: Scope determines variable visibility: global, function, and block. var is function-scoped and hoisted; let/const are block-scoped and not initialized until declaration (TDZ).

<script>
console.log(typeof hoistedVar); // "undefined"
var hoistedVar = 5;

// let/const example:
try {
  console.log(blockLet); // ReferenceError (TDZ)
} catch(e) {}
let blockLet = 2;
</script>

10. Closures

Description: A closure is a function that retains access to its lexical environment even after outer function returns — great for private state and factories.

<script>
function counter(){
  let count = 0;
  return function(){
    count++;
    return count;
  };
}
const c = counter();
console.log(c()); //1
console.log(c()); //2
</script>

11. Arrays & Common Methods

Description: Arrays hold lists. Useful methods: push/pop, shift/unshift, map, filter, reduce, forEach, find.

<script>
let arr = [1,2,3];
arr.push(4);
let evens = arr.filter(n => n%2===0);
let sum = arr.reduce((acc,n) => acc + n, 0);
console.log(evens, sum);
</script>

12. Objects & Prototypes

Description: Objects are collections of properties. JS objects have prototypes — inheritance is prototype-based. You can add shared behavior via prototypes (or classes which wrap prototypes).

<script>
// object literal
let person = { name: "Ramesh", age: 20 };
// accessing
console.log(person.name);

// prototype example
function Animal(type){
  this.type = type;
}
Animal.prototype.speak = function(){ console.log(this.type + " speaks"); };
let a = new Animal("Dog");
a.speak(); // "Dog speaks"
</script>

13. Classes & OOP

Description: ES6 classes are syntactic sugar over prototypes — they define constructors, methods, and support inheritance with extends and super(). Use classes for clear OOP structure.

<script>
class Person {
  constructor(name, age){
    this.name = name;
    this.age = age;
  }
  greet(){
    return `Hi, I'm ${this.name}`;
  }
}
class Student extends Person {
  constructor(name, age, roll){
    super(name, age);
    this.roll = roll;
  }
  info(){ return `${this.name} (roll ${this.roll})`; }
}
const s = new Student("Ramesh", 20, 101);
console.log(s.greet(), s.info());
</script>

14. DOM Manipulation

Description: The DOM is a tree of nodes representing the page. Use querySelector, getElementById, modify textContent, innerHTML, attributes, classes and styles.

<script>
const el = document.querySelector("#title");
el.textContent = "Updated title";
el.classList.add("highlight");
</script>
<h3 id="title">Original</h3>

15. Events & Event Listeners

Description: Respond to user actions with event listeners (addEventListener). Use event objects to inspect the target and control behavior (preventDefault, stopPropagation).

<script>
document.querySelector("#btn").addEventListener("click", function(e){
  e.preventDefault();
  console.log("Button clicked", e.target);
});
</script>
<button id="btn">Click</button>

16. Event Delegation

Description: Attach a single handler on a parent to manage many children — efficient for dynamic lists and improves performance.

<script>
document.querySelector("#list").addEventListener("click", (e) => {
  const li = e.target.closest("li");
  if(li) console.log("Clicked item:", li.textContent);
});
</script>
<ul id="list"><li>a</li><li>b</li></ul>

17. Fetch API & AJAX

Description: Fetch is a modern promise-based API to request resources. It replaces older XHR usage. Responses often need .json() or .text().

<script>
fetch("https://jsonplaceholder.typicode.com/todos/1")
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));
</script>

18. Promises

Description: Promises represent async results. States: pending → fulfilled or rejected. Chain with .then(), handle errors with .catch().

<script>
const p = new Promise((resolve, reject) => {
  setTimeout(()=> resolve("done"), 500);
});
p.then(msg => console.log(msg)).catch(err => console.error(err));
</script>

19. Async / Await

Description: Syntactic sugar on top of promises — allows writing asynchronous code that looks synchronous. Use try/catch for errors.

<script>
async function getData(){
  try{
    const res = await fetch("https://jsonplaceholder.typicode.com/todos/1");
    const json = await res.json();
    console.log(json);
  } catch(err){ console.error("Fetch error", err); }
}
getData();
</script>

20. Error Handling

Description: Use try / catch / finally to handle runtime errors. Throw custom errors with throw when input or state is invalid.

<script>
try{
  throw new Error("Something went wrong");
} catch(e){
  console.error(e.message);
} finally {
  console.log("Always runs");
}
</script>

21. JSON (Data Interchange)

Description: JSON is a text format for structured data. Use JSON.stringify() to serialize and JSON.parse() to deserialize.

<script>
const obj = {name:"Ramesh", age:20};
const str = JSON.stringify(obj);
const back = JSON.parse(str);
console.log(back.name);
</script>

22. LocalStorage & SessionStorage

Description: Browser storage: LocalStorage persists across sessions, SessionStorage clears on tab close. Store strings (serialize objects to JSON).

<script>
localStorage.setItem("user","Ramesh");
console.log(localStorage.getItem("user"));
sessionStorage.setItem("temp","123");
</script>

23. Timers

Description: Use setTimeout for delayed execution and setInterval for repeated tasks. Always clear intervals when not needed.

<script>
const id = setInterval(()=> console.log("tick"), 1000);
setTimeout(()=> clearInterval(id), 5000); // stop after 5s
</script>

24. Regular Expressions

Description: RegEx are patterns for string matching and validation (emails, formats). Use /pattern/flags with test and match.

<script>
const re = /^[a-z0-9]+@[a-z0-9]+\.[a-z]{2,}$/i;
console.log(re.test("user@example.com"));
</script>

25. Modules (ES Modules)

Description: Split code into modules. Use export / import. Modules are loaded with type="module" in script tags (or via bundlers).

<!-- module.js -->
<script type="module">
export function sum(a,b){ return a+b; }
</script>

<!-- main.js -->
<script type="module">
import {sum} from './module.js';
console.log(sum(2,3));
</script>

26. Browser APIs

Description: Browsers expose APIs: Geolocation, Canvas (2D/3D drawing), History/Location, Notifications, Clipboard and many more to build richer apps.

<script>
// Geolocation
if(navigator.geolocation){
  navigator.geolocation.getCurrentPosition(pos => console.log(pos.coords));
}
// Canvas example (create a circle)
// <canvas id="c" width="200" height="200"></canvas>
const c = document.getElementById('c');
if(c){ const ctx = c.getContext('2d'); ctx.fillStyle='orange'; ctx.beginPath(); ctx.arc(100,100,50,0,Math.PI*2); ctx.fill(); }
</script>

27. Performance & Optimization

Description: Optimize reflows (batch DOM changes), debounce/throttle heavy events, use requestAnimationFrame for animations, minimize heavy loops, and lazy-load resources.

<script>
// debounce example
function debounce(fn, wait=200){
  let t;
  return (...args) => { clearTimeout(t); t = setTimeout(()=> fn(...args), wait); };
}
window.addEventListener('resize', debounce(()=> console.log('resized')));
</script>

28. Security Basics

Description: Sanitize user input to prevent XSS, understand CORS for cross-origin requests, use HTTPS, and avoid exposing secrets in client code.

<!-- prevent XSS: never insert untrusted HTML -->
<script>
const userInput = "<img src=x onerror=alert(1)>";
const textNode = document.createTextNode(userInput);
document.body.appendChild(textNode); // safe
</script>

29. Testing & Debugging

Description: Use browser DevTools (breakpoints, console, network), unit tests (Jest), and linters (ESLint). Console logs help but use debuggers for better inspection.

<script>
// Simple debugging
const obj = {a:1, b:2};
console.table(obj);
debugger; // pause execution if DevTools open
</script>

30. Best Practices & Next Steps

Description: Write modular code, keep functions small, handle errors gracefully, document public APIs, use version control, learn a bundler (Webpack/Vite), and practice building projects.
