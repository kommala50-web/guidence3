1. Introduction to SQL

Description: SQL (Structured Query Language) is used to interact with relational databases. It allows you to query, insert, update, and delete data efficiently. SQL is standardized but may have dialect differences (MySQL, PostgreSQL, MSSQL).

-- Simple SELECT query
SELECT * FROM Customers;

2. Database Concepts

Description: Understand databases, tables, rows, columns, primary keys, foreign keys, indexes, and relationships between tables (1:1, 1:N, N:M).

-- Example Table
CREATE TABLE Customers(
    CustomerID INT PRIMARY KEY,
    Name VARCHAR(50),
    Email VARCHAR(50)
);

3. Data Types

Description: Learn SQL data types: INT, FLOAT, DECIMAL, VARCHAR, CHAR, TEXT, DATE, DATETIME, BOOLEAN, etc. Each type has size and constraints.

CREATE TABLE Products(
    ProductID INT,
    Name VARCHAR(100),
    Price DECIMAL(10,2),
    InStock BOOLEAN
);

4. Basic Queries

Description: Use SELECT to retrieve data, specify columns, use aliases, and ORDER BY for sorting.

SELECT Name, Price AS Cost
FROM Products
ORDER BY Price DESC;

5. Filtering Records

Description: Use WHERE clause with conditions and logical operators (AND, OR, NOT) to filter results.

SELECT * FROM Customers
WHERE Country='USA' AND Age>25;

6. Pattern Matching

Description: LIKE operator allows pattern matching with % (any sequence) and _ (single character).

SELECT * FROM Customers
WHERE Name LIKE 'A%'; -- Names starting with A

7. Aggregate Functions

Description: COUNT, SUM, AVG, MIN, MAX are used to summarize data. Often combined with GROUP BY.

SELECT COUNT(*) AS TotalOrders, AVG(Total) AS AvgAmount
FROM Orders
GROUP BY CustomerID;

8. GROUP BY & HAVING

Description: GROUP BY aggregates data by column. HAVING filters aggregated results.

SELECT CustomerID, SUM(Total) AS TotalSpent
FROM Orders
GROUP BY CustomerID
HAVING SUM(Total)>1000;

9. JOINs

Description: Combine tables using INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN.

SELECT Orders.OrderID, Customers.Name
FROM Orders
INNER JOIN Customers
ON Orders.CustomerID = Customers.CustomerID;

10. Subqueries

Description: A query inside another query. Can be used in SELECT, FROM, WHERE clauses.

SELECT Name
FROM Customers
WHERE CustomerID IN (SELECT CustomerID FROM Orders WHERE Total>500);

11. Set Operations

Description: Combine results from multiple queries with UNION (distinct), UNION ALL, INTERSECT, EXCEPT.

SELECT Name FROM Customers_USA
UNION
SELECT Name FROM Customers_Canada;

12. Modifying Data

Description: Use INSERT to add rows, UPDATE to modify, DELETE to remove. Always backup important data.

INSERT INTO Customers(Name, Email) VALUES('Sita','sita@example.com');
UPDATE Customers SET Email='new@example.com' WHERE Name='Sita';
DELETE FROM Customers WHERE Name='Sita';

13. Transactions & ACID

Description: Ensure data integrity with transactions. Use COMMIT to save and ROLLBACK to undo. ACID = Atomicity, Consistency, Isolation, Durability.

BEGIN TRANSACTION;
UPDATE Accounts SET Balance=Balance-100 WHERE ID=1;
UPDATE Accounts SET Balance=Balance+100 WHERE ID=2;
COMMIT;

14. Indexes

Description: Indexes speed up queries. Types: Single-column, composite, unique, full-text. Trade-off: slower inserts/updates.

CREATE INDEX idx_customer_name
ON Customers(Name);

15. Views

Description: Virtual tables based on queries. Simplify complex queries and restrict access.

CREATE VIEW TopCustomers AS
SELECT CustomerID, SUM(Total) AS TotalSpent
FROM Orders
GROUP BY CustomerID
HAVING SUM(Total)>1000;

16. Stored Procedures

Description: Predefined SQL code blocks stored in the database. Helps reuse logic and improve performance.

CREATE PROCEDURE GetTopCustomers()
BEGIN
  SELECT CustomerID, SUM(Total) AS TotalSpent
  FROM Orders
  GROUP BY CustomerID
  ORDER BY TotalSpent DESC;
END;

17. Functions

Description: User-defined functions return values and can be scalar or table-valued. Use them for reusable logic.

CREATE FUNCTION GetDiscount(Total DECIMAL)
RETURNS DECIMAL
BEGIN
  RETURN Total*0.1;
END;

18. Triggers

Description: Automated actions triggered by events (INSERT, UPDATE, DELETE). Useful for auditing or enforcing rules.

CREATE TRIGGER AuditInsert
AFTER INSERT ON Orders
FOR EACH ROW
BEGIN
  INSERT INTO OrdersAudit(OrderID, CreatedAt)
  VALUES(NEW.OrderID, NOW());
END;

19. Constraints

Description: Ensure data validity: PRIMARY KEY, FOREIGN KEY, UNIQUE, CHECK, NOT NULL, DEFAULT.

CREATE TABLE Employees(
    ID INT PRIMARY KEY,
    Name VARCHAR(50) NOT NULL,
    Salary DECIMAL CHECK(Salary>0)
);

20. Normalization

Description: Organize tables to reduce redundancy. Normal forms 1NF, 2NF, 3NF, BCNF. Helps maintain data integrity.

-- Example: split CustomerInfo table into Customers & Addresses

21. Denormalization

Description: For performance, combine tables to reduce joins. Trade-off: redundant data and update anomalies.

-- Example: store total orders in Customers table

22. Advanced Queries

Description: Use CTEs (WITH), window functions (ROW_NUMBER, RANK), CASE statements for complex analytics.

WITH RecentOrders AS (
  SELECT *, ROW_NUMBER() OVER(PARTITION BY CustomerID ORDER BY OrderDate DESC) AS rn
  FROM Orders
)
SELECT * FROM RecentOrders WHERE rn=1;

23. Window Functions

Description: Perform calculations across a set of rows related to current row (SUM(), AVG(), RANK(), ROW_NUMBER()).

SELECT Name, Salary, RANK() OVER(ORDER BY Salary DESC) AS Rank
FROM Employees;

24. Cursors

Description: Used to iterate row-by-row over query results in stored procedures.

DECLARE emp_cursor CURSOR FOR
SELECT Name FROM Employees;
OPEN emp_cursor;
-- FETCH and process each row
CLOSE emp_cursor;

25. Transaction Isolation Levels

Description: Control how concurrent transactions interact. Levels: READ UNCOMMITTED, READ COMMITTED, REPEATABLE READ, SERIALIZABLE.

SET TRANSACTION ISOLATION LEVEL SERIALIZABLE;

26. Performance Tuning

Description: Use indexes, query optimization, EXPLAIN plan, partitioning, and caching to improve performance.

EXPLAIN SELECT * FROM Orders WHERE Total>100;

27. Backup & Restore

Description: Regularly backup databases. Methods: SQL dump, binary backups, point-in-time recovery.

-- MySQL backup
mysqldump -u user -p dbname > backup.sql
-- restore
mysql -u user -p dbname < backup.sql

28. Security & Permissions

Description: Use GRANT, REVOKE, roles, and least privilege principles to secure databases.

GRANT SELECT, INSERT ON Employees TO 'readonly_user'@'localhost';
REVOKE INSERT ON Employees FROM 'readonly_user'@'localhost';

29. Stored Procedures Advanced

Description: Use input/output parameters, error handling, loops, and dynamic SQL in procedures.

CREATE PROCEDURE UpdateSalary(IN empID INT, IN raise DECIMAL)
BEGIN
  UPDATE Employees SET Salary=Salary+raise WHERE ID=empID;
END;

30. SQL Functions Advanced

Description: Built-in functions: string, date/time, numeric, conversion. Combine with CASE and COALESCE.

SELECT Name, COALESCE(ManagerID,0) FROM Employees;

31. OLAP & Analytical Queries

Description: Use GROUPING SETS, CUBE, ROLLUP for multi-dimensional analytics in SQL.

SELECT Region, Product, SUM(Sales)
FROM Sales
GROUP BY CUBE(Region, Product);

32. JSON & XML

Description: Modern databases support JSON/XML columns. Use JSON functions to query nested structures.

SELECT data->'$.name' AS Name
FROM CustomersJSON;

33. NoSQL Intro

Description: Learn differences between SQL and NoSQL (document, key-value, column, graph) databases.

-- Example: MongoDB query
db.users.find({age: {$gt:20}});

34. SQL Best Practices

Description: Use consistent naming, comments, indexes wisely, normalize properly, avoid SELECT *, and test queries.

-- Add comment
-- This query fetches active customers
SELECT Name FROM Customers WHERE Active=1;

35. Transactions Advanced

Description: Savepoints, nested transactions, error handling, rollback strategies.

SAVEPOINT sp1;
UPDATE Accounts SET Balance=Balance-50 WHERE ID=1;
ROLLBACK TO sp1;

36. Partitioning

Description: Split large tables horizontally or vertically for performance and manageability.

CREATE TABLE Orders_2025 PARTITION BY RANGE(OrderDate);

37. Replication & HA

Description: Master-slave replication, clustering, failover strategies to ensure uptime.

-- MySQL replication setup example

38. CTE & Recursive Queries

Description: Recursive CTEs allow hierarchical queries like org charts or bill of materials.

WITH RECURSIVE Subordinates AS (
  SELECT ID, ManagerID FROM Employees WHERE ManagerID IS NULL
  UNION ALL
  SELECT e.ID, e.ManagerID
  FROM Employees e
  INNER JOIN Subordinates s ON e.ManagerID=s.ID
)
SELECT * FROM Subordinates;

39. SQL Optimization

Description: Analyze query plans, avoid unnecessary joins, index columns used in WHERE, and denormalize if needed.

EXPLAIN SELECT * FROM Orders WHERE CustomerID=1;
